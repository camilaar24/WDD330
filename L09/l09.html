Ch14: HTML5 APIs

Data- attributes, introduced in HTML5, allow you to define custom data associated with DOM elements in a standardized way. Going back to HTML4, the data related to DOM elements were stored in other attributes, such as class, id, or rel, depending on the developer's preferences.

The value of the data-* attribute can be any text string. It can contain numbers but the data type is a string. For example, data-age="30" would have the text value 30, which is different from the number 30.

The Geolocation API allows us to find out the user's position, in terms of geographic coordinates of latitude and longitude, as long as they give permission to share that information with the website. It is implemented with the geolocation object, which is a property of the navigator object.

Web Workers allows scripts to be executed in the background, without being interrupted by events produced by user interactions with the interface.

Background scripts are recommended for tasks that can be expensive, in terms of duration and/or resource consumption since they can run in parallel to other scripts that are being executed on the page.

Canvas, this API provides us with a rectangle on our web page in which we can draw anything we imagine using JavaScript, even providing animation and interactivity. To do this, the API exposes us to a set of methods or functions that we can use to define and create shapes, gradients, paths or apply various transformations.

Ch12: Canvas, SVG, and Drag and Drop

A canvas is a screen-resolution-dependent bitmap canvas that can be used to render graphics, game images, or any other such information on the fly. The canvas is a new HTML5 element that allows us to draw graphics with Javascript. It helps us to represent texts, images, graphics of all kinds, rectangles, lines, gradients, and effects dynamically on the screen. Drawing on the canvas is done through the Canvas 2D API. This API contains a multitude of functions with which we can practically draw anything on it.

"canvas id="myCanvas" style="width:300px; height:300px">/canvas"

SVG or Scalable Vector Graphics is a web standard for defining vector-based graphics on web pages. Based on XML, the SVG standard provides markup to describe paths, shapes, and text within a viewport. Markup can be embedded directly into HTML for display or saved to an .svg file and inserted like any other image.

You can write SVG by hand, but more complicated graphics can be designed in vector graphics editors like Illustrator or InkScape and exported to SVG files or code.

Developers initialize an "svg" graph with the XML namespace and tag like this:

svg xmlns="http://www.w3.org/2000/svg" version="1.1"

With a viewport in place, you can add basic graphics, text, and path elements.
