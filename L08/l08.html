Getting Started with JSON Web Tokens (JWT)

Transform

JWT (JSON Web Token) is a standard that is contained in the RFC 7519 document. It defines a mechanism to be able to propagate between two parties, and in a safe way, the identity of a certain user, in addition to a series of claims or privileges.

These privileges are encoded in JSON-type objects, which are embedded within the payload or body of a message that is digitally signed.

JWTs consist of three parts separated by dots (.), which are:

EXAMPLE: { "alg": "HS256", "type": "JWT" }

Payload: where the user data and privileges appear, as well as all the information that we want to add, and all the data that we deem convenient.

EXAMPLE: { "sub": "1234567890", "name": "John Doe", "admin": true }

Signature: a signature that allows us to verify if the token is valid, and this is where the crux of the matter lies, since if we are trying to make secure communication between parties and we have seen that we can take any token and see its content with a simple tool.

EXAMPLE: HMACSHA256( base64UrlEncode(header) + "." + base64UrlEncode(payload), secret)

JWT always invites us to communicate between parties using HTTPS to encrypt the traffic, so that if someone were to intercept it, the traffic itself through HTTP over those SSL sockets would encrypt all the communication, the token, and everything. the rest, and thus add that possibility of security
