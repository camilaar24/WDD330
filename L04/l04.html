How Single-Page Applications Work

SPAs are pages that are always served from the same file, that is, they have to be downloaded before they can be executed. This has an improvement in terms of user experience since the user has the feeling of navigating between pages very quickly.

This has an improvement in terms of user experience since the user has the feeling of navigating between pages very quickly.

The Problem with Single Page Apps

A SPA does not have pages, it has views, and although we see that the URL changes, this is not really the case, since everything is served from the same page.

This means that today, indexing robots, such as Google's, have problems executing them, since they are not able to do it correctly and display all the HTML content, so they are not able to understand it and index them correctly.

These types of pages are always executed on the client-side, with their good and negative things, since they improve the user experience, but, in the case of having a slow connection, downloading that page to access it can be a bit more tedious.
