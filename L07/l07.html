Ch8: Transforms and Transitions

Transform

If you only want to move an element vertically(transform: translateX(45px);) or horizontally (transform: translateY(-30px);) you can use the translatex or translatey functions respectively.

Transforms don’t work on inline elements. Just add display: inline-block; to the span.

The scale(x,y) function scales an element by the defined factors horizontally then vertically.

transform: scale(1.5, 0.25);

As with translate, you can also use the scaleX(x) or scaleY(y) functions.: transform: translateX(40px) scale(1.5);

Transitions

The transition-property property defines the CSS properties of the element that should be transitioned, with all for all properties being the default.

The transition-duration property sets how long the transition will take: the duration of time it takes to go from the default state to the transitioned state: transition-duration: 0.2s;

The transition-timing-function lets you control the pace of the transition in even more granular detail: transition-timing-function: ease-out;

With the transition-delay property, it’s possible to introduce a delay before the transition begins. Normally a transition begins immediately, so the default is 0. Include the number of milliseconds (ms) or seconds (s) to delay the transition: -webkit-transition-delay: 50ms; transition-delay: 50ms;

Animations

To create an animation, use the @keyframes rule for IE10+ and FF16+. Include @-webkit-keyframes for all WebKit implementations followed by a name of your choosing, which will serve as the identifier for the animation. Then, you can specify your keyframes.

Propierties:

  • animation-name: appearDisappear;
  • animation-duration: 300ms;

    • animation-timing-function: linear;

  • animation-iteration-count: infinite;
  • animation-direction: alternate;
  • animation-delay: 50ms;
  • animation-fill-mode: both;

    Callbacks

    Callbacks can be used to facilitate event-based asynchronous programming. JavaScript executes code sequentially in descending order. However, there are some cases where code is (or should be) executed after something else starts, and also non-sequentially. This is called asynchronous programming.

    Callbacks ensure that a function will not be executed before a task completes, but will be executed just after the task has completed. It helps us develop asynchronous JavaScript code and keeps us safe from issues and bugs.

    Closures

    Closures are widely used in JavaScript, you just have to know how to recognize them within the code and the functionality implications that it has.

    Closures are available because the language implements lambdas and higher-order functions, and they are a direct consequence of writing code using lexical scopes. Closure is a confusing concept because its use is "invisible". Unlike other concepts such as functions, variables or others. Closures are not always used consciously and directly.
