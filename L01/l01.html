Managing the Modern Frontend Workflow

Package managers are programs that keep track of all of our app's external dependencies. This includes any development tools or libraries that we may use. It not only knows which packages to download, but it also keeps track of their versions. For our package manager, we use npm. Bundlers are responsible for the compilation, translation, concatenation, minification, and movement of assets in our project. As a bundler, we're utilizing Snowpack. Parcel and Webpack are two other popular bundlers. Task managers keep track of what has to be done and when it needs to be done. In most cases, scripts will be defined in the task management for each development phase. We're using npm again for task management because our project is quite straightforward.

Git it Together

Git is a tool that performs a function of code version control in a distributed way, of which several features stand out, such as:

  • Quick branch management, because Git predicts that a change would be merged far more frequently than it was originally planned.
  • Distributed management; Changes are imported as extra branches, which can be merged in the same way that the local branch can.
  • Project management that is effective.
  • Package replenishing on a regular basis.
    • There are two types of Version Control Systems, Centralized and Distributed. Centralized Version Control Systems (VCS), such as Subversion, which is a tool that has been trusted to house version revision history, is a centralized point, which can lead to a loss of work if we lose network connectivity. Distributed Version Control Systems (DVCS) get around this problem. Some examples of distributed systems, apart from Git, are Mercurial, Bazaar, or Darcs. In these types of tools, clients fully replicate the repository.
